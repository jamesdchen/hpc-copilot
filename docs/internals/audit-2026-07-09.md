### R1 ‚Äî Re-home non-workflow strays from `ops/` root into subjects

Rule (from `docs/architecture.md` + `scripts/lint_subject_imports.py`):
`ops/` root holds workflow-verb primitives (cross-subject orchestrators,
exempt from the cross-subject lint because they sit outside subject
dirs); everything else lives inside a subject dir.

1. Enumerate every `src/hpc_agent/ops/*.py` root module and classify:
   `grep -o 'verb="[a-z]*"' <file>` ‚Äî `verb="workflow"` stays at root;
   any other verb (or helper with no `@primitive`) is a candidate stray.
2. For each candidate, verify moveability BEFORE moving: its
   `hpc_agent.*` imports must be only its own future subject +
   `infra.*`/`state.*` substrate (the cross-subject lint's allowlist).
   A stray importing a *different* subject is actually an orchestrator ‚Äî
   leave it at root and note it.
3. Known-verified moves: `attention_op.py`, `attention_queue.py`,
   `attention_render.py` ‚Üí new subject `ops/attention/` (they import
   only each other + substrate; `tests/ops/attention/` already exists).
   `notebook_gate.py`, `notebook_status.py`, `notebook_view.py` ‚Üí
   `ops/notebook/` (they import only `ops.notebook.*` + substrate).
   NAME COLLISION WARNING: `ops/notebook/view_op.py` already exists and
   is a different module from root `notebook_view.py` ‚Äî pick
   non-colliding leaf names and keep the registered primitive names
   (the CLI verb string) unchanged.

   (`scripts/lint_subject_init.py` enforces this).
5. Update every importer: `grep -rln 'ops\.<old_name>' src/ tests/
