# Repo audit — 2026-07-09 (organization + bug sweep)

> **Fix status:** every finding below was fixed on this branch in the
> commit series following the audit commit (`fix:` / `fix(<area>):`
> subjects), each with a pinning test. Two audit claims were corrected
> during fixing: (1) the transport tar-fallback `mv` failure was masked
> by a worse bug — the no-rsync preclean's `find | xargs rm -rf`
> deleted the protected `.hpc` runtime files through the `-path`
> prunes on every re-push; both are fixed. (2) The
> `slash_commands = ["agents/*.md"]` package-data glob is deliberate,
> not stale: `agent_assets` installs an optional `agents/` tree and
> core currently ships none (§6 worker removal). Deferred by choice:
> the larger organization restructures (ops/ flat-vs-dir layout,
> `slash_commands` package rename, CHANGELOG split, docs/design
> status split) are recommendations, not fixes — they belong in
> their own reviewed change.

Ten parallel auditors swept every subsystem (`infra`, `state`, `ops` ×3,
`execution` + cluster-side templates, `_kernel`, `cli`/`_wire`/schemas,
`experiment_kit`/`incorporation`/`meta`/`recovery`, `scripts`/CI/packaging/
`slash_commands`/`examples`). Findings below passed the
"can this actually fire on a real path" bar from
`docs/internals/engineering-principles.md`; each carries a concrete failure
scenario. Verification status: **[C]** = confirmed by the coordinating
reviewer against the source; **[R]** = reproduced empirically by the
subsystem auditor (live execution in the repo venv); **[P]** = plausible,
traced in source but not executed.

## Part 1 — Organization

1. **Root `settings.json` is inert.** Claude Code loads project settings from
   `.claude/settings.json`; there is no `.claude/` directory, so the committed
   safety policy (scancel/qdel/rm -rf denies, ruff/mypy PostToolUse hook) never
   applies, while README.md:97 cites it as a live guarantee. Move it (and
   `settings.local.json.example`) under `.claude/`.
2. **Merge `tests/contract/` and `tests/contracts/`.** Same purpose, split
   5 vs 39 files; the `contract` pytest marker registered in
   `tests/contract/conftest.py` is scoped to that dir only, so
   `pytest -m contract` misses the 39 tests in `tests/contracts/`.
3. **`ops/` mixes layouts** — 12 subject directories plus ~40 loose
   subject-prefixed root modules (`ops/notebook_status.py` beside
   `ops/notebook/`; `ops/attention_*.py` with no `ops/attention/`, while
   `tests/ops/attention/` exists). Fold loose modules into their subject dirs
   or document the flat-vs-dir rule.
4. **Top-level `slash_commands` package** installs unnamespaced into
   site-packages (collision landmine). Move under `hpc_agent/` or rename.
   (The `slash_commands = ["agents/*.md"]` package-data glob matching
   nothing is deliberate — `agent_assets` installs an optional `agents/`
   tree and core ships none since the §6 worker removal.)
5. **CHANGELOG.md is 352 KB** — archive old majors under `docs/changelog/`.
6. **Dev deps declared twice** — `[project.optional-dependencies].dev` and
   `[dependency-groups].dev` both exist (both pin pytest-xdist). Consolidate.
7. **`docs/design/` mixes durable architecture with one-shot plans/retros**
   (`*-implementation-plan.md`, `proving-run-*-hardening.md`). Add status
   frontmatter or a `history/` subfolder.
8. Small: `src/hpc_agent/integration/` is a 1-file package; `ci/` holds only
   the SLURM docker harness and would sit better next to its consumer.

## Part 2 — Bugs

### Tier 1 — silent result corruption / lost cluster work

- **[C] `src/hpc_agent/ops/memory/interview.py:785`** — `_materialize_tasks_py`
  merges `{**_TASKS[i], **_INJECT}`, so `fixed_params`/frozen-sha constants
  OVERRIDE swept axis values — the opposite of the wire contract
  (`_wire/actions/interview.py` `_FIXED_PARAMS_DESC`: "a swept axis of the same
  name wins") and of the adjacent comment's stated intent. A cartesian sweep
  over `samples=[100,200,300]` with `fixed_params={"samples":10000}` runs every
  task at 10000; task counts and cmd_sha checks all pass. Found independently
  by two auditors; reproduced.
- **[C] `src/hpc_agent/execution/mapreduce/dispatch.py:924`** — the
  stdlib-only cluster dispatcher does
  `from hpc_agent.ops.recover.service import inject_service_env` when
  `HPC_SERVICE_ENV` is set, but deploy ships no `hpc_agent` package
  (`transport._build_deploy_items`). Any run with a `service_env` block fails
  every array task with `ModuleNotFoundError` before the executor spawns.
  The self-containment contract test only exercises `--help`, so this
  function-local import escapes it.
- **[R] `src/hpc_agent/execution/mapreduce/combiner.py:384-437`** — the wave
  combiner re-imports `.hpc/tasks.py` and calls `tasks.resolve(tid)` instead of
  reading the sidecar's frozen `trial_params` (which dispatch.py:648 treats as
  ground truth). Any state-dependent `resolve()` — including the shipped
  `optuna_strategy.py`/`pbt_strategy.py` scaffolds — combines against the NEXT
  iteration's kwargs: wrong result_dirs, empty partials written silently, or a
  phantom optimizer trial minted on the login node.
- **[R] `src/hpc_agent/experiment_kit/axis_matcher/_ast_utils.py:181-217`** —
  two classifier holes: (a) a read-modify-write accumulator nested inside
  `if`/`with`/`try` (`if x > 0: total += x`) is judged loop-local →
  `independent`, auto-recorded by `classify-axis-auto` branch D with no LLM
  review → parallel chunking silently loses the accumulation; (b) mutating
  method calls other than `.append` (`.add`, `.update`, `.extend`) are not
  stores, so `seen.add(x)` + `len(seen)` classifies `independent` the same way.
- **[R] `src/hpc_agent/experiment_kit/axis_matcher/matchers/window.py:24-77`**
  — the bounded-window matcher never checks the carried set: a loop with a
  `deque(maxlen=W)` append PLUS an unbounded prefix-sum commits `bounded_halo`,
  so chunked execution computes the sum only over chunk+halo.
- **[R] `src/hpc_agent/execution/mapreduce/reduce/status.py:774-777,1000-1004`**
  — tasks classified complete-by-result-file drop the scheduler's `job_info`,
  so `reduce_resource_usage` counts nothing for them: a fully complete run
  reports `cpu_hours: 0, tasks_counted: 0` exactly when it consumed the most
  compute. The existing test covers only RUNNING tasks.
- **[C] `src/hpc_agent/infra/backends/query.py:464` (SGE), `:307` (PBS)** —
  qacct/`qstat -f` dedup keeps the FIRST (oldest) job's row per task, while
  `ops/recover/runner.py:155` promises newest-wins ordering (SLURM implements
  it at :225). A task that failed in job A and succeeded in resubmitted job B
  is permanently reported failed on SGE/PBS.
- **[P] `src/hpc_agent/infra/backends/_engine.py:711-735` +
  `infra/cluster_logs.py:79-96`** — log-path construction uses the GLOBAL task
  id, but waved submissions submit LOCAL `1-<size>` arrays with `TASK_OFFSET`;
  over-cap runs fetch the WRONG task's stderr (global 1005's log returned for
  task 5) or "missing" for wave≥1 tasks, mis-fingerprinting failures.
- **[P] `src/hpc_agent/infra/backends/query.py:203-205`** — `query_sacct`
  skips every row without `_` in JobID, dropping all rows for non-array MPI
  jobs (#293 submits `array=False`), so a failed MPI job has no accounting
  state.
- **[P] `src/hpc_agent/execution/mapreduce/reduce/metrics.py:74-109` +
  `combiner.py:120-148`** — when executors omit `n_samples`, per-wave partials
  lose group size and the cross-wave reduce weights waves equally: a grid
  point split 9-tasks/1-task across a wave boundary gives the lone task 9× its
  correct weight.
- **[P] `src/hpc_agent/execution/mapreduce/dispatch.py:792-797`** — the
  cmd_sha-changed re-run path leaves the previous experiment's `metrics.json`
  in a shared result_dir; if the new attempt fails or is still running, status
  and combiner count the stale file as this run's completed result.

### Tier 2 — lost state / broken workflows

- **[R] `src/hpc_agent/state/runs.py:525-534`** — `find_existing_runs` matches
  any `.json`, sweeping up the `<run_id>.<block>.terminal.json` records that
  `state/block_terminal.py:169` writes into the same `.hpc/runs/` dir:
  (1) `prune_orphan_sidecars` — called with `min_age_seconds=0` on every
  `submit_flow_batch` — deletes all prior runs' terminal records; (2)
  `find_run_by_cmd_sha` can return a terminal record (same cmd_sha, newer
  mtime), which reads back jobless → dedup falls through and re-submits work
  the cluster is already running; (3) terminal files count toward the
  `MAX_RUNS` retention cap, evicting real sidecars.
- **[R] `src/hpc_agent/ops/submit/runner.py:518-573`** — the A5 cmd_sha dedup
  fallback misfires post-qsub because the crash-safety pre-stamp (added
  2026-06-11) invalidated the orphan guard's "own sidecar is still jobless"
  assumption: every `invalidate_on_code_change=True` submit skips the initial
  watchdog stamp (`next_tick_due=None` → invisible to `find_stalled_runs`);
  in a campaign with an older complete same-params run, the new qsub'd run gets
  NO journal record while the old run's journal is overwritten `in_flight`.
- **[C] `src/hpc_agent/cli/submit.py:110-126`** — batch-shaped specs route to
  `cmd_submit_flow_batch` BEFORE `--partial-ok`/`--invalidate-on-code-change`

  batch verb has no equivalents.
- **[R] `src/hpc_agent/_kernel/lifecycle/block_drive.py:73,523-526`** — the
  resume tick strips only `next_block` from the committed `resolved`, but the
  decision journal sanctions identity echoes (`cmd_sha`, `run_id`) and every
  block spec is `extra="forbid"` — a resumed span fails `extra_forbidden`
  precisely on the inputs the §4 fast-path requires; retried identically every
  tick.
- **[R] `src/hpc_agent/_kernel/lifecycle/block_drive.py:506-508`** — the
  pending-decision marker is cleared BEFORE the resumed span runs; any span
  failure destroys the resume cursor, and a failed `rerun` replays as
  `advance`, dropping the human's nudge.
- **[R] `src/hpc_agent/_kernel/lifecycle/block_drive.py:413-425`** — the `{}`
  fresh-entry fallback is applied to `aggregate-check` (requires `run_id`), so

  documented clear skip.
- **[R] `src/hpc_agent/ops/monitor/kill.py:162-165`** — `settled=True` is set
  on any non-raising reconcile return, including the `unable_to_verify` path

- **[R] `src/hpc_agent/meta/campaign/atoms/acknowledge_budget.py:132-165`** —

- **[R] `src/hpc_agent/meta/campaign/atoms/load_context.py:433-436`** — the

- **[R] `src/hpc_agent/infra/transport.py:518-531`** — the tar-fallback
