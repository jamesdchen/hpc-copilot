# Live conformance: the chart judges, the operator adjusts

Live conformance extends the attestation substrate into operation time — SPC
(Shewhart's control chart) rebuilt on this repo's primitives. A registration is a
hypothesis; production is the experiment that never stops. The substrate
OBSERVES, JUDGES, and ROUTES; every actuation — halting a strategy, recalibrating
an instrument, placing an order — is the operator's act, OUTSIDE this system
entirely. The full design center is `docs/design/live-conformance.md` (settled
2026-07-07); these rows hold the boundary its enforcement-row table names, the
**no-actuation pin first-class** (the agency boundary, mechanized).

## Enforcement map

| Rule | Enforced by | Fires when |
|---|---|---|
| **NO ACTUATION AFFORDANCE (first-class — the agency boundary mechanized):** the ONLY conformance mutate verb is `conformance-record` (sole side effect: one ledger append); no core conformance code path reaches a broker/instrument/external system (no network/subprocess import, no credential field in `state/conformance*` + `ops/conformance*`); a `nonconforming` verdict changes NO registration status, revokes nothing, halts nothing — drift routes, never acts (live-conformance C1/C2, the agency boundary) | `tests/contracts/test_conformance_boundary.py::test_only_conformance_mutate_verb_is_record_with_a_single_append`, `::test_conformance_modules_reach_no_external_system`, `::test_no_credential_field_in_conformance_modules`, `::test_nonconforming_window_leaves_registration_journal_byte_identical` | a halt/pause/recalibrate/deploy affordance lands, a conformance module imports a network/subprocess/device client, or a nonconforming window mutates the registration journal — the substrate reaching past the chart into the machine |
| Observation receipts route through the ONE attestation kernel — the append binds via `state/attestation.py::bind` (the payload sha is server-recomputed, never trusted from the wire); the comparator is the ONE definition `state/conformance.py::judge_window`, which routes its envelope through the shared `determinism.py` order-statistics helper, never a re-inlined min/max (live-conformance C-store/C-compare) | `tests/contracts/test_conformance_boundary.py::test_append_binds_through_the_attestation_kernel`, `::test_comparator_is_the_one_definition`; `tests/ops/decision/test_conformance_authorship.py::test_verdict_fabricated_receipt_sha_refused` | a receipt path bypasses `bind`, a consumer re-implements the comparator, or a second envelope reduction appears |
| The baseline is SEALED and FIXED — it resolves only against a dossier-sealed artifact (a declared baseline whose sha no manifest entry carries is refused at the registration append), and live receipts never enter it (recording N receipts changes no baseline byte; no admission path — re-baselining is re-registration) (live-conformance C-declare/C-compare) | `tests/contracts/test_conformance_boundary.py::test_baseline_not_sealed_in_dossier_refused_at_registration`, `::test_recording_receipts_changes_no_baseline_byte`; `tests/ops/decision/test_conformance_authorship.py::test_baseline_not_in_manifest_refused` | a "learning limits" / rolling-baseline branch lands, or a baseline the dossier does not seal passes the registration gate — production drift laundered into the limits that exist to catch it |
