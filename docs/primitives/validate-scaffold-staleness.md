# validate-scaffold-staleness

Pre-submit gate that refuses a `.hpc/` scaffold generated by an **older** hpc-agent before it fails with a runtime `ImportError` on a compute node (#364). A scaffold built under a prior version can survive an upgrade and break far downstream — the observed case was a pre-reorg `.hpc/_build_tasks.py` importing `from hpc_agent.template import ...` after `hpc_agent.template` was consolidated away. hpc-agent already version-stamps sidecars / manifests / journal records; this validator closes the same gap for the **generated scaffold**: `build-tasks-py` stamps the generating `hpc_agent.__version__` into `.hpc/.scaffold_meta.json`, and this atom checks that stamp (and the scaffold's imports) against the installed package at submit time — locally, with **no SSH**.

The check is a **no-op when the stamp matches the installed version**: it returns `fresh` without scanning a single import, so a current-version scaffold submits with byte-identical behavior.

## Inputs

- `experiment_dir` (path, required) — Repo root. The validator reads `<experiment_dir>/.hpc/.scaffold_meta.json` (the generator stamp) and the generated scaffold files (`tasks.py`, `cli.py`, and any legacy `_build_tasks.py`). No `--spec` — the experiment dir is the only input.

## Outputs

A `ValidateScaffoldStalenessResult` object with:

- `findings` (list of `ValidatorFinding`) — Empty list = pass. When the scaffold is stale, a single error finding is emitted with `code="stale_scaffold"`, a message naming the version mismatch, and a `suggested_fix` pointing at regeneration (re-run onboarding / `build-tasks-py --force`). Its `evidence` carries `installed_version`, `stamp_version`, `legacy_artifacts`, `unresolved_imports`, and `scanned_files`.
- `status` (`"fresh"` | `"stale"`) — The classification even when there is no finding, so a caller can tell "fresh because the stamp matched" from "fresh because every import resolved."
- `installed_version` (string) — The installed `hpc_agent.__version__` the scaffold was checked against.
- `stamp_version` (string | null) — The generator version stamped in `.hpc/.scaffold_meta.json`, or `null` for an unstamped (legacy) scaffold.

## Errors

None declared on the primitive. The validator never raises on a stale or malformed scaffold — it surfaces the problem via `findings` (the validator contract). The `validate-campaign` composer escalates any error-severity finding to `overall="fail"`, which aborts submit.

## Idempotency

Pure local read of `<experiment_dir>/.hpc/`; calling twice with the same on-disk state produces the same result. No SSH, no journal mutation.

## Notes

- **No-op fast path.** When the scaffold's `generator_version` stamp equals the installed version (and no legacy `_build_tasks.py` is present), the validator returns `fresh` immediately without scanning any import — guaranteeing a current-version scaffold is unaffected.
- **Mismatch alone is not stale.** A stamp from a different version whose imports all still resolve against the installed package is **not** refused; only a vanished module / removed symbol — or a pre-reorg `_build_tasks.py` artifact — flags `stale`. This keeps the gate from false-refusing across benign version bumps.
- **Unknown generator → verify, don't refuse.** An unstamped (legacy) scaffold is not silently trusted nor automatically refused: its `hpc_agent.*` imports are scanned against the installed package, and it is flagged stale only if one no longer resolves.
- **Legacy `_build_tasks.py` is stale by construction.** Current generators emit `.hpc/tasks.py`; the presence of a pre-reorg `_build_tasks.py` is a dead generated artifact from a prior version and is flagged regardless of whether its imports happen to resolve.
- **Regenerate, don't hand-edit.** The stale file is framework-generated, not user source — the `suggested_fix` is to regenerate the framework-owned scaffold (re-run onboarding via `hpc-wrap-entry-point` / `interview`, or `hpc-agent build-tasks-py --force`), never to hand-edit the superseded file.
- **Composed by `validate-campaign`** and run **unconditionally** (it needs no spec field, only the experiment dir) — a stale scaffold fails on the cluster regardless of which other inputs the caller supplied.
