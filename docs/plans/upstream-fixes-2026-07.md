### G1. pull-status-over-silent-channels — RANK 1

- **Thesis:** run/task status is PULLED over long-lived, mostly-silent SSH
  channels (client polls, remote walks), so every middlebox, reaper,
  quota, and latency property of a silent channel becomes a run-state
  failure mode.
- **Fired symptoms retired:** r12-f24 (NAT idle-drop severs every
  long-silent leg at ~100s; the 40-minute reporter walk; misattributed
  twice), r12-f20 (login-node fork exhaustion from orphaned remote halves
  of severed channels; self-service-unrecoverable at worst), r12-f19 (S3
  watch read ssh timeouts as run death, fired a terminal harvest at a live
  array), r12-f16 (detached S2 staging/liveness blindness — zero liveness
  during a 14-min wedge), r12-f3 (no liveness surface on the MCP link;
  2×~10-min wedges). Contributory/overlap: #8 (the idle reaper was the
  second severing mechanism — primary G4), #3 (the 25–40-min silent
  reporter walk the channel must survive — primary G13).
- **Diagnosis cost (cited):** the 2026-07-11 saga — "one generator, a full
  day" (audit doc); r12-f24: "misattributed twice before the evidence
  converged" (first to fork exhaustion, then to the stale wheel); measured
  control run 40m26s wall for <2 CPU-minutes; r12-f3: ~10 min of human
  attention ×2 with zero mechanical disclosure.
- **Upstream alternative:** crash-only monitoring / cluster-announces — the
  cluster ANNOUNCES state transitions (task markers in ONE directory, one
  readdir) instead of the client pulling status over a channel that must
  stay alive; the question stops being asked over a channel at all.
  Phase 1 shipped the same day the cluster diagnosed it. Belt fixes
  already landed under it: keepalives in ssh_argv, remote-deadline wrapper
  (orphans self-destruct), #8 inflight fix, frozen-manifest reporter,
  stray-sweep verb.
- **Migration cost:** medium — announce kernel (cluster-side markers +
  reader) and watch consumption remain; the belt layer is landed. Phase 1
  shipped 2026-07-11.
- **Flip trigger (scope-by-constraint):** pull-over-ssh was the ship-time
  call (no cluster-side agent, works on day one against any cluster). The
  flip is recorded and partially fired: announce kernel replacing the pull
  reporter for the heavy walk (r12-f24: "the announce kernel … is the fix
  that carries the weight"); steady state when the watch consumes
  announcements end-to-end (post-run-13 window).
- **Confidence:** high (the user-recorded exemplar; membership verified).

- **Confidence:** high (the dockets name the generator themselves; 13+
  independent symptoms).

### G4. hand-rolled-connection-lifecycle (vs library-native) — RANK 4

- **Thesis:** the framework re-implements transport-lifecycle mechanisms
  its libraries already own (idle management, lease/TTL bookkeeping,
  keepalive policy); each hand-rolled mechanism mints failure modes no
  library-native stack has.
- **Fired symptoms retired:** #8 (idle reaper measures idleness from last
  COMPLETION → severs any >120s in-flight command, then silently
  re-executes it one-shot — duplicate execution of non-idempotent remote
  commands; this was the SECOND severing mechanism of the 2026-07-11
  night), #35 (ssh_slots: fixed TTL below the worst legitimate hold +
  ownership-blind release → cascading over-admission of the host the
  limiter protects), r12-f16 (stale detach `lease.lock` with a blocking,
  timeout-less acquire freezes every successor worker), r12-f24
  library-boundary lesson (asyncssh's native keepalives/lifecycle were
  CORRECT; the one hand-rolled piece layered on top is where the sever
  lived).
- **Diagnosis cost (cited):** r12-f24 addendum — a second severing
  mechanism had to be hunted the same night, with keepalives already
  active ("misattributed twice"); r12-f16: 14+ min frozen worker + the
  forensics to prove 0.015s CPU; #8's duplicate-side-effect hazard
  (combiner writes, slow qsub) is unbounded.
- **Upstream alternative (ruled, B16):** shrink hand-rolled lifecycle to
  what NO library can know — the ban-risk breaker and connection-RATE
  courtesy are cluster-social POLICY and stay; the MECHANISMS (idle
  management, keepalives, multiplexing, lease/TTL/pid bookkeeping) are
  the library's (asyncssh; the filelock/psutil outsourcing precedent for
  leases and slots). ssh_slots is the named next suspect.
- **Migration cost:** medium — delete/outsource the reaper class of code,
  re-home slot leases on a library lease; the #8 inflight-counter belt is
  already landed.
- **Flip trigger (recorded in r12-f24):** the native-binary-first verdict
  was a SHIP-TIME call (Windows named-pipe agent, engine maturity); the
  steady-state verdict is engine-first with library-native lifecycle —
  "the trigger that flips it is the #8 inflight fix + this lesson landing
  in a released wheel." Ruled post-run-13 (ruling 5 confirms schedule).
- **Confidence:** high.

- **Confidence:** high.
